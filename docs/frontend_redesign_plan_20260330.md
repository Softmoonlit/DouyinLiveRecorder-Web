# 前端重设计冻结计划（2026-03-30）

## 一、目标
在不破坏现有后端接口与已通过验收脚本的前提下，完成 Web 前端三大标签页重设计：

1. 直播录制页：高还原参考项目交互与信息密度。
2. 看板页（Phase 3）：高还原参考项目的交互层级与视觉表达。
3. 设置页（Phase 4）：以本项目配置模型为唯一准绳，适配优先，不强制还原参考项目设置结构。

## 二、总体策略

1. 技术路线：Vue3 + Naive UI + Vite 前端子工程。
2. 对接方式：通过 API 适配层连接现有 FastAPI 接口。
3. 数据刷新：保持 3 秒轮询（仅在标签激活时执行），本期不引入 SSE。
4. 风险控制：后端最小改动，保持 scripts/phase5_regression.sh、scripts/phase6_config_validation.sh、scripts/phase6_network_validation.sh 可持续通过。

## 三、阶段规划

### Phase 0 - 契约冻结与差异映射（起点）
1. 冻结当前接口与字段：/api/v1/tasks、/api/v1/dashboard、/api/v1/config/settings、/api/v1/config/update-settings、/api/v1/summary。
2. 输出“参考交互 -> 本项目接口”映射表，区分可复用项、降级项、补强项。
3. 保持状态语义不变：offline|monitoring|live_not_recording|recording|stopping|failed 及其 live_status/recording_status 映射。

### Phase 1 - 前端骨架与设计系统（依赖 Phase 0）
1. 建立三主标签页信息架构：直播录制、看板、设置。
2. 建立统一 token：颜色、间距、阴影、圆角、状态色、动效。
3. 建立响应式规范：桌面（>=1080）与移动（<1080）双断点。

### Phase 2 - 直播录制页高还原实现（依赖 Phase 1）
1. 实现筛选、排序、分页、列表视图、列显隐与批量操作。
2. 实现新增/编辑/批量操作弹窗与统一反馈机制（toast + 字段级错误）。
3. 持久化页面偏好：分页大小、排序字段、列配置。
4. 本期不实现卡片视图及视图切换入口，统一采用列表视图。

### Phase 3 - 看板页高还原实现（依赖 Phase 1，可与 Phase 2 并行）
1. 高还原参考项目看板结构：头部刷新、指标卡矩阵、快捷操作区、结果反馈弹窗。
2. 高还原视觉表达：图标渐变、hover 层级、告警态动画、卡片层次。
3. 三列状态看板：正在直播 / 正在录制 / 未开播，支持跳转联动。
4. 数据字段以本项目可用能力为准：任务总数、启用/禁用、录制中、失败、平台分布、运行时长、磁盘空间（按后端可用性接入）。
5. 刷新策略保持 3 秒轮询，离开标签自动暂停。

### Phase 4 - 设置页适配重构（依赖 Phase 1，可与 Phase 2/3 并行）
1. 采用 schema 驱动渲染，复用后端 group/batch/type/choices/min/max/effect 元数据。
2. 分组以本项目配置批次为主：第一批录制核心、第二批网络稳定、第三批通知与凭据预留位。
3. 明确不强制还原参考项目设置结构：不新增后端未支持字段，不引入伪配置项。
4. 强化校验与生效提示：保存前校验、保存后字段级反馈、支持重新读取。
5. 保持数值兼容：int 字段允许 300.0 等整值浮点输入并归一化为整数。

### Phase 5 - 最小后端补强（依赖 Phase 0，可与 Phase 2/3/4 并行）
1. 仅补齐支撑看板高还原展示的必需指标。
2. 建议优先新增：应用运行时长、磁盘空间摘要。
3. 对无法稳定提供的参考专有指标，在 UI 明确降级说明。
4. 保持现有任务与配置接口兼容，不影响既有回归基线。

### Phase 6 - 集成切换与回退策略（依赖 Phase 2/3/4/5）
1. 引入前端版本开关（例如 ?ui=v2 或环境变量）用于灰度切换。
2. 保留旧版页面回退入口，直到新界面验收完成。
3. Docker 构建链加入前端打包产物，继续由 FastAPI 托管静态资源。

### Phase 7 - 验证与验收（依赖 Phase 6）
1. 自动化回归：
   - scripts/phase5_regression.sh
   - scripts/phase6_config_validation.sh
   - scripts/phase6_network_validation.sh
2. 交互验收矩阵：
   - 录制页：筛选、排序、分页、批量、弹窗、列显隐（仅列表视图）。
   - 看板页：高还原结构、视觉、告警态、轮询与跳转联动。
   - 设置页：读取、保存、重载、字段级错误与生效提示。
3. 设备验收：桌面 Chrome + 移动窄屏触控操作可用。

## 四、关键文件

1. index.html：现有入口与旧版回退基线。
2. app.py：静态资源托管与路由入口。
3. src/runtime/api_manager.py：看板聚合与状态映射扩展点。
4. src/runtime/config_service.py：设置 schema 与字段归一化逻辑。
5. Dockerfile：前端构建与静态产物打包入口。
6. docker-compose.yaml：运行参数与灰度开关透传。
7. scripts/phase5_regression.sh：后端功能回归。
8. scripts/phase6_config_validation.sh：配置行为回归。
9. scripts/phase6_network_validation.sh：网络稳定性回归。

## 五、冻结决策

1. 录制页：高还原实现。
2. 看板页（Phase 3）：高还原实现。
3. 设置页（Phase 4）：项目适配优先，不强制还原参考项目设置结构。
4. 刷新机制：维持 3 秒轮询，本期不引入 SSE。

## 六、验收红线

1. 设置页只展示后端 schema 返回字段，禁止出现本项目未支持的参考配置项。
2. 不破坏任务 CRUD、启停、看板与设置接口的既有契约。
3. 变更后必须可通过既有 Phase 5/6 回归脚本。

## 七、范围外事项

1. 多用户权限系统。
2. 历史录制统计系统重建。
3. 复杂图表中心（仅预留扩展位，不纳入首批）。
