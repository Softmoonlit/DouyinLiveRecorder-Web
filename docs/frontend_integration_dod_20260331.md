# 前后端联调检查单（DoD，2026-03-31）

## 1. 目的与范围

本检查单用于将重设计拍板规则转化为可执行、可记录、可签字的联调门槛，覆盖以下范围：

1. 前端版本切换与灰度回退。
2. 录制页、看板页、设置页核心交互口径。
3. 后端接口口径与前端展示一致性。
4. Phase 5/6 回归脚本通过性。

## 2. 关联文档

1. docs/frontend_redesign_plan_20260330.md（第九章拍板执行规则）。
2. docs/phase5_regression.md。
3. docs/phase6_config_validation.md。
4. docs/phase6_network_validation.md。

## 3. 角色与责任

1. 前端：页面行为、状态渲染、交互反馈、移动端可用性。
2. 后端：接口契约、字段口径、错误语义、可用性标记。
3. 测试：用例执行、缺陷闭环、回归结论。
4. 产品/负责人：拍板项确认与最终签字。

## 4. 进入联调门槛

1. 已合入拍板规则（1C 2C 3C 4B 5C 6C 7C）。
2. 联调环境可访问以下接口：
   - /api/v1/tasks
   - /api/v1/dashboard
   - /api/v1/config/settings
   - /api/v1/config/update-settings
   - /api/v1/summary
3. 至少准备一组可复现数据：
   - 覆盖 offline、monitoring、live_not_recording、recording、stopping、failed。
   - 覆盖至少 2 个平台与跨分页任务数量。

## 5. DoD 检查项（必须逐项记录）

### DOD-01 版本开关决策矩阵（对应 1C）

检查内容：

1. URL 参数优先于会话记忆、环境变量与默认值。
2. URL 传入值不在允许集合时，必须回退，不可生效。
3. 页面内切换版本后，刷新页面仍保留会话版本。
4. 新界面存在“切回旧版”入口。

通过标准：

1. 三条最小验收用例全部通过。
2. 无“同一会话内版本随机切换”问题。

### DOD-02 状态映射与跳转联动（对应 2C）

检查内容：

1. recording -> 正在录制。
2. live_not_recording -> 正在直播。
3. monitoring、offline -> 未开播。
4. stopping -> 正在录制，且显示“停止中”子态。
5. failed 按子状态纠偏：
   - recording_status 为 recording 或 stopping 时归入正在录制。
   - 否则 live_status 为直播中时归入正在直播。
   - 其他归入未开播。
6. failed 必须带异常高亮，不新增第四列。
7. 点击三列跳转录制页时，只带列过滤条件，不叠加隐式筛选。

通过标准：

1. 状态真值表覆盖样例全部正确。
2. 跳转后的筛选条件可复现、可解释。

### DOD-03 3 秒轮询状态机（对应 3C）

检查内容：

1. 采用请求完成后再调度下一次的模型。
2. 标签切回激活态立即首刷。
3. 单飞约束生效：同接口同标签不出现并发在途请求。
4. 离开标签页后暂停，返回后恢复。
5. 连续失败退避：3s -> 6s -> 12s -> 24s，上限 30s。
6. 成功后退避恢复到 3s。

通过标准：

1. 抓包可见无重复并发请求。
2. 错误重试节奏符合退避规则。

### DOD-04 设置页提交语义（对应 4B）

检查内容：

1. 仅提交脏字段。
2. effect 触发的自动赋值字段会被纳入提交。
3. 隐藏字段仅在本次被联动改写时提交。
4. 部分失败时返回字段级错误，成功字段状态不回滚。
5. 保存成功后自动重读并以服务端值回填。
6. int 字段接受整值浮点输入（如 300.0）并归一化为整数。

通过标准：

1. 提交载荷与变更字段一致。
2. 字段级错误可定位到具体控件。
3. 配置保存后前后端显示一致。

### DOD-05 录制页批量操作边界（对应 5C）

检查内容：

1. 默认仅支持当前页全选。
2. 支持二次确认后升级为当前筛选结果全选。
3. 筛选条件变化后，跨页选择自动清空并提示。
4. 高风险操作（停录、删除）必须二次确认，文案包含影响条数。
5. 部分成功反馈格式统一为“成功 X 条，失败 Y 条”，并可展开失败明细。

通过标准：

1. 无“用户不知自己在操作哪一批任务”的歧义。
2. 部分成功时可追溯失败对象与原因。

### DOD-06 看板指标口径字典（对应 6C）

检查内容：

1. 后端返回原始值、单位、采样时间戳、可用性标记。
2. 运行时长展示为 d h m。
3. 磁盘展示为已用/总量 + 使用率，单位 GiB，保留 1 位小数。
4. available=false 时显示 --，并提供“指标暂不可用”说明。

通过标准：

1. 同一时刻接口数据与页面展示可一一对应。
2. 指标不可用时不出现误导性 0 值。

### DOD-07 移动端列表显示规范（对应 7C）

检查内容：

1. 断点生效：>=1080（桌面）、768-1079（平板窄窗）、<768（手机）。
2. 手机仅展示 P0 列，P1/P2 进入行展开区。
3. 平板展示 P0+P1，P2 进入行展开区。
4. 触控区域不小于 44x44 px。
5. 优先使用行展开，不将横向滚动作为默认主路径。

通过标准：

1. 手机窄屏下核心操作可单手完成。
2. 不出现核心状态被挤出首屏的问题。

### DOD-08 回归脚本准入门槛

检查内容：

1. scripts/phase5_regression.sh 通过。
2. scripts/phase6_config_validation.sh 通过。
3. scripts/phase6_network_validation.sh 通过。

通过标准：

1. 三个脚本退出码均为 0。
2. 未出现阻断级回归缺陷。

## 6. 执行顺序建议

1. 先做 DOD-01、DOD-02、DOD-03（框架行为）。
2. 再做 DOD-04、DOD-05（核心交互）。
3. 再做 DOD-06、DOD-07（展示口径与移动端）。
4. 最后做 DOD-08（回归准入）。
